## 2. 청구항 3 (보정): 웜홀 라우팅 시스템

### 2.1 기술 구성

독립된 가족 박물관 도메인 간 **연합 인증(Federation Authentication)**을 통해
개인정보를 최소한으로 노출하면서 가족 관계 기반 데이터 접근을 가능하게 하는 시스템.

### 2.2 핵심 구성요소

1. **비대칭 키 인프라**
   - 각 가족 박물관 도메인은 RSA-2048 키 쌍을 자동 생성
   - 공개키는 `domain_public_keys` 테이블에 평문 저장
   - 개인키는 AES-256-CBC로 암호화하여 저장 (scrypt 파생 키 사용)
   - 키 교환은 연합 요청 수락 시 자동 수행

2. **RS256 JWT 기반 연합 토큰**
   - 발신 도메인의 개인키로 서명 (RS256 알고리즘)
   - 수신 도메인은 발신 도메인의 공개키로 검증
   - 토큰 페이로드: `iss`(발신 도메인), `sub`(요청 사용자 ID), `federationId`, `nonce`(1회용)
   - 토큰 유효기간: 5분 (단기)

3. **Nonce 기반 재사용 공격 방지**
   - 각 연합 요청에 UUID v4 nonce 포함
   - 사용된 nonce는 `nonce_cache` JSONB 배열에 저장 (최근 100개 유지)
   - 동일 nonce 재사용 시 즉시 거부 (HTTP 403)

4. **범위 기반 접근 제어 (Scope Validation)**
   - 관계 유형별 기본 범위:
     - 직계(direct): `[profile, photos.public, photos.family, exhibitions]`
     - 방계(collateral): `[profile, photos.public]`
     - 배우자(spouse): `[profile, photos.public, exhibitions]`
   - 요청 범위가 합의 범위를 초과하면 거부 (scope escalation 방지)
   - 합의 범위는 양 도메인 소유자가 승인 시 확정

5. **데이터 조회 흐름 (Resolve)**
   ```
   Source Domain → POST /api/federation/token (사용자 JWT로 Fed JWT 발급)
                 → GET /api/federation/resolve/:fedId/:nodeId (Fed JWT로 데이터 조회)
                 ← Target Domain: 범위 내 인물 데이터 반환
   ```

### 2.3 종래 기술과의 차별점

| 구분 | 종래 기술 (OAuth2 Federation) | 본 발명 (웜홀 라우팅) |
|------|------|------|
| 인증 방식 | 중앙 IdP 의존 | 도메인 간 직접 RSA 키 교환 |
| 범위 제어 | OAuth scope (범용) | 가족 관계 유형별 세분화 scope |
| 재사용 방지 | token expiry만 의존 | nonce + expiry 이중 방어 |
| 키 관리 | 중앙 서버 관리 | 각 도메인 자체 AES-256 암호화 |
| 합의 과정 | 관리자 일방 설정 | 양 도메인 소유자 상호 승인 |

## 3. 청구항 4 (신규): 디지털 유산 승계 시스템

### 3.1 기술 구성

가족 박물관 소유자의 계정 미활동을 감지하여, 미리 지정된 승계자에게
박물관 관리 권한을 자동 또는 수동으로 이전하는 시스템.

### 3.2 핵심 구성요소

1. **승계자 등록**
   - 사이트 소유자만 승계자 등록/수정/삭제 가능
   - 승계자 정보: 이름, 이메일, 관계, 활성화 조건
   - 복수 승계자 등록 가능 (우선순위 순)

2. **활성화 조건**
   - `inactivity_1year`: 소유자 1년 미로그인 시 자동 활성화
   - `inactivity_2year`: 소유자 2년 미로그인 시 자동 활성화
   - `manual`: 소유자가 직접 활성화 버튼 클릭

3. **승계 프로세스**
   ```
   소유자 마지막 로그인 → 미활동 기간 경과 → 승계자에게 이메일 알림
   → 승계자가 이메일 링크 클릭 → 본인 인증 → 박물관 관리 권한 이전
   ```

4. **안전장치**
   - 승계 활성화 전 소유자에게 경고 이메일 발송 (30일, 7일, 1일 전)
   - 소유자가 경고 기간 내 로그인하면 승계 타이머 자동 리셋
   - 승계 완료 후에도 원래 소유자가 로그인하면 되돌리기 가능 (30일 이내)

5. **데이터 보존**
   - 승계 시 모든 데이터(사진, 가족트리, 전시, 방명록) 100% 보존
   - 소유자 변경 로그 기록 (감사 추적)

### 3.3 종래 기술과의 차별점

| 구분 | 종래 기술 (Google Inactive Account Manager) | 본 발명 |
|------|------|------|
| 대상 | 개인 계정 데이터 | 가족 단위 박물관 (멀티 멤버) |
| 승계 범위 | 데이터 다운로드/삭제만 | 관리 권한 이전 (운영 지속) |
| 관계 기반 | 없음 | 가족 관계 기반 승계 우선순위 |
| 되돌리기 | 불가 | 30일 이내 되돌리기 가능 |
| 알림 체계 | 단일 알림 | 3단계 사전 경고 (30/7/1일) |

## 4. 청구항 5 (신규): 조상 기억 시스템

### 4.1 기술 구성

가족트리의 인물 데이터와 조상 기억 기록을 양방향으로 연동하여,
다차원적 생애 기록(사진, 이력, 회고록, 방명록)을 보존하는 시스템.

### 4.2 핵심 구성요소

1. **가족트리 연동 (person_id 링크)**
   - 조상 기억 레코드를 가족트리의 특정 인물(person)에 연결
   - 가족트리에서 인물 클릭 → 해당 조상 기억 페이지로 이동
   - 조상 기억에서 연결된 인물의 가계 정보 자동 표시

2. **다차원 기록 구조**
   - **대표 사진** (featured_photos): 최대 3장, 1+2 레이아웃 (메인 1장 + 서브 2장)
   - **이력** (biography): 학력, 경력, 주요 이력 텍스트
   - **회고록** (memoir): 생애 이야기, 추억, 업적
   - **방명록** (guestbook): 가족/방문자의 추모 메시지

3. **3-Photo Featured Layout**
   ```
   ┌──────────┬──────────┐
   │          │  Photo 2 │
   │ Photo 1  ├──────────┤
   │ (Main)   │  Photo 3 │
   └──────────┴──────────┘
   ```
   - 1장: 전체 너비
   - 2장: 좌우 50/50
   - 3장: 좌 50% (메인) + 우 50% (상하 분할)

4. **공개 범위 제어**
   - `public`: 누구나 열람 가능
   - `family`: 가족 멤버만 열람 가능
   - 카드에 공개 범위 뱃지 표시

5. **CRUD 지원**
   - 추가/수정/삭제 모두 소유자/멤버만 가능
   - 삭제 시 확인 팝업 표시

### 4.3 종래 기술과의 차별점

| 구분 | 종래 기술 (Legacy.com, Ancestry.com) | 본 발명 |
|------|------|------|
| 트리 연동 | 별도 시스템 | 가족트리 인물과 양방향 링크 |
| 사진 레이아웃 | 단일 썸네일 | 3-Photo Featured Layout |
| 데이터 소유 | 플랫폼 소유 | 사용자 자체 도메인 보유 |
| 승계 | 불가 | 디지털 유산 승계와 통합 |
| 방명록 | 있음 | 있음 + 가족 멤버 권한 분리 |

## 6. 발명의 효과

1. **데이터 주권 보장**: 각 가족 도메인이 독립적으로 데이터를 소유하며,
   연합 인증을 통해 필요한 범위만 상호 공유

2. **세대 간 지속성**: 디지털 유산 승계를 통해 가족 기록이 소유자의
   부재 시에도 영속적으로 보존됨

3. **프라이버시 최소 노출**: scope 기반 접근 제어와 nonce 방어로
   불필요한 데이터 노출 원천 차단

4. **가족 관계 기반 지능형 제어**: 직계/방계/배우자 등 관계 유형에 따라
   차등화된 접근 범위 자동 적용
