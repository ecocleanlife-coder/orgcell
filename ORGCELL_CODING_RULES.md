# Orgcell 가족유산박물관 — 코딩 표준 규칙서

**작성일:** 2026-04-05
**버전:** v3.0 (최종 확정)
**원칙:** 본 문서의 설계 원칙은 모든 코드 구현의 최우선 순위임.

---

## 1. 서비스 정체성

- 한국어: **가족유산박물관**
- 영어: **Family Heritage Museum**
- 태그라인: "FamilySearch가 죽은 자의 도서관이라면, Orgcell은 산 자의 박물관입니다."
- "디지털" 단어 사용 금지

---

## 2. 블록 규격 (레고 표준)

- 카드 = 220px × 220px
- 간격 = 20px (부부 내부 제외)
- 그리드 = 240px (카드 220 + 간격 20)
- 세대 간격 = 280px (카드 220 + 위아래 간격 60)
- 부부 카드 간격 = 0px (붙어있음)
- 부부 박스 = 440px (카드 220 × 2)
- 선 굵기 = 4px

---

## 3. 가족나무 배치 4원칙

1. 남자 왼쪽, 여자 오른쪽 (항상)
2. 내 배우자는 내 옆에
3. 내 형제는 내 옆에, 배우자 형제는 배우자 옆에
4. 나머지 배우자들의 가문(부모/형제)은 평면에 안 보이지만 데이터로 존재 → 클릭하면 펼쳐짐

---

## 4. 블록 표시

- 사진, 이름, ID, 대표정보 (최대 3줄)
- 호버 시 "OOO 박물관" 툴팁만
- 호버 메뉴 없음
- 고인도 동일 표시, 생존기간만 (예: 1930~2005)

---

## 5. 블록 입체감

**모든 블록 — 약한 입체감:**
```css
.folder-card {
  border: 1px solid #C4A882;
  background: #FAFAF5;
  border-right: 2px solid #b09060;
  border-bottom: 2px solid #9a7a50;
  box-shadow: 2px 2px 0 #c4a87a;
}
```

**관장 부부 — 강한 입체감:**
```css
.folder-card.curator {
  border: 2px solid #8B7355;
  background: #FDF8F0;
  border-right: 4px solid #9a7a50;
  border-bottom: 4px solid #7a6040;
  box-shadow: 3px 3px 0 #c4a87a, 6px 6px 0 #b09060;
}
```

## 6. 클릭 동작

- 클릭 → 해당 인물 박물관 방문 (4원칙으로 트리 새로 구성)
- 박물관 없는 사람 클릭 → 가족나무 + "아직 박물관 오픈 전입니다" + 임시 부모 생성 + "무료로 시작하기" 유도
- 관장 더블클릭 → 해당 인물 자료실 진입
- 🏠 버튼 → 원래 박물관으로 복귀

---

## 7. ID 체계

- 형식: 국가코드-5자리영문숫자랜덤 (예: KR-A3K7B)
- 국가코드 기준: 한국어→KR, 영어→접속지 IP (US, GB 등)
- 혼동 방지: O/0, I/1, L 제외
- 사용 문자: 2-9, A-H, J-N, P-Z (30자)
- 부모/자녀 생성 시 부모 국가코드 상속
- 카드에 이름 아래 ID 표시

### 좌측 — 인물 정보:
```
[사진] [사진불러오기]  이름: [___]
                      ID: KR-A3K7B
마우스/손가락으로      생년월일: [___]
크기·위치 조정         ☐ 사망 → 사망일: [___]
                      대표정보1: [___]
                      대표정보2: [___]
                      대표정보3: [___]

[가족 추가]  [저장]
```

### 우측 — 메뉴 버튼 (세로 배치):
```
[사진자료실]     → 공개 시 "사진전시관"
[주요자료실]     → 공개 시 "자료전시관"
[주요약력]       → 공개 시 "약력전시관"
[자서전]         → 공개 시 "자서전전시관"
[작품실]         → 공개 시 "작품전시관"
[육성녹음]       → 공개 시 "음성전시관"
[공유앨범]       → 생성 즉시 상단 메뉴에 노출
─────────────
[초대하기]
[접근요청관리]
```

### 하단:
- 우측 버튼 클릭 시 해당 내용 표시
- 미클릭 시 비어있음

## 10. 공개 → 전시관 자동 생성

- 자료실에서 공개 체크된 자료는 박물관 상단 메뉴에 전시관으로 자동 추가
- 공개 자료가 없는 전시관은 메뉴에 안 나타남

## 11. 공유앨범

### 생성 (자료실 → [공유앨범] 클릭 → 하단):
```
새 공유앨범 만들기

제목: [2025 하와이 가족여행_______]
공개범위: ○ 전체공개 ○ 가족만 ○ 링크만

[사진 불러오기]  [초대하기]  [생성]

─── 내 공유앨범 목록 ───

■ 2025 하와이 가족여행    📸 234장
  [공유링크 복사] [초대하기] [삭제]

■ 2025 NCI Cancun        📸 567장
  [공유링크 복사] [초대하기] [삭제]
```

### 공유앨범 전시관 (상단 메뉴에서 클릭 시):
- 로그인 없이 사진 업로드 + 다운로드 가능
- 각 사진마다 체크박스 → 선택 다운로드
- 중복 사진 자동 감지 + 제거
- 사진 올리기 시 "보유하고 계신 사진을 정리해 드릴까요?" 안내
- 생성 즉시 박물관 상단 메뉴에 노출
- 하단에 Orgcell 가입 유도

## 12. 상단 메뉴 스타일

- 자판기 버튼 스타일 (둥근 모서리, 입체감, 누르면 2px 내려감)
- 기존 메뉴: 금색 톤 (#FDF8F0 배경, #C4A882 테두리)
- 공유앨범: 푸른 톤 (#F0F5FD 배경, #8BA5C4 테두리), 📷 아이콘 통일
- 메뉴가 많아지면 자연스럽게 다음 줄로 넘어감

---

## 13. 조상전시관

- 각 사람이 본인 박물관에 추가 가능
- 자료는 조상의 자료실에 먼저 기록 (원본)
- 원하는 것을 선택하여 본인 조상전시관에 전시
- 상속인이 비공개 시 다른 사람 전시관에서도 자동으로 내려감

---

## 14. 권한 체계

- 관장 → 모든 인물 자료실 접근 가능
- 관장이 등록한 인물 → 당사자 계정 만들기 전까지 관장 전체 권한
- 당사자 계정 생성 후 → 당사자가 주인
- 외부인 클릭 → AccessDeniedModal → 관장에게 요청

---

## 15. 디지털 유산 승계

- 망자의 박물관 관장 권한은 유언 또는 법적 상속인이 승계
- 공개 권한도 상속인이 가짐
- 상속인이 자료 추가/수정/삭제 가능

---

## 16. 프라이버시 3레이어

- PUBLIC: 이름, 관계, 공개 사진
- FAMILY: 생년월일, 추가 사진 (8촌 기본)
- PRIVATE: 비공개 메모, 개인정보

## 22. 가족나무 표시 범위

### 화면에 표시하는 것
1. 나 + 배우자
2. 내 자녀 + 그 자손들 (무한 아래로)
3. 내 형제 + 그 자녀 + 그 자손들
4. 배우자 형제 + 그 자손들
5. 내 부모 + 그들의 형제 (삼촌/이모)
6. 배우자 부모 + 그들의 형제
7. 내 조부모
8. 배우자 조부모

**→ 조부모(depth=2)까지만 표시. 조부모 형제(depth=2 비직계)는 숨김.**

### 화면에 표시하지 않는 것
- 증조부모 이상 (depth ≥ 3): DB 보관, 화면 숨김, 자료실에서만 열람 가능
- 조부모의 형제 (depth=2 중 직계 조부모 아닌 노드)

### 코드 상수
```js
const DISPLAY_MAX_ANCESTOR_DEPTH = 2; // 조부모까지
```

## 23. 배치 및 선연결 룰

### 간격 룰
- 카드와 카드 사이: 20px
- 부부 박스와 박스 사이: 20px
- 모든 블록(카드/박스) 사이: 20px 통일
- 부부 내부만 0px

### 부부
- 부부 박스로 묶음 (수평선 아님)
- 남자 왼쪽, 여자 오른쪽
- 부부 center-to-center = CARD_W = 220px

### 부모→자녀 연결선
- 부부 박스 아래 중간에서 수직선 내림
- 수직선이 자녀들의 수평 연결선 중간에 닿음

### 여러 자녀
- 각 자녀 위 중간에서 형제간 수평선 연결
- 수평선 중간에서 위로 수직선 → 부모 박스 연결
- 자녀 순서: 생년월일 오름차순 (장남/장녀부터) 좌→우

### 자녀에 배우자 생기면
- 자녀 옆에 남좌여우로 배치, 부부 박스로 묶임
- 그만큼 좌우 공간 확장

### 배우자 부모 연결선 (CRITICAL)
- 부부 박스 중앙이 아닌 **해당 배우자 카드 개인 중앙**에서 선 올라감
- 예: 임윤님 부모 선은 "이순호─임윤님" 박스 중앙이 아닌 임윤님 카드 중앙에서 올라감

### 형제 배치
- 남편 형제 → 남편 왼쪽으로 240px씩
- 아내 형제 → 아내 오른쪽으로 240px씩
- 형제에 배우자 생기면 옆에 배치 + 박스 + 공간 확장

### 화면 중앙 배치
- 초기 화면: 로그인한 사람(부부) 부부 박스 중심이 화면 가로 정중앙
- 타 박물관 방문 시: 해당 인물(부부) 박스 중심이 화면 가로 정중앙
- 🏠 복귀 시: 로그인한 사람 부부 박스 중심이 화면 가로 정중앙

## 21. 검색 기능

박물관 초기 화면 상단에 검색바:
```
[🔍 이름 또는 ID로 검색 _______________]
```

- 이름 검색: 한글/영문 모두 지원
- ID 검색: KR-A3K7B 형식으로 직접 검색
- 검색 결과: 해당 인물의 박물관으로 이동
- PUBLIC 정보만 검색 가능 (PRIVATE 인물은 검색에 안 나옴)
- 거절자(RefusedPersonBox)는 검색 결과에서 제외
