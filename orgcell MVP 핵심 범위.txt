## 1. 프로젝트 개요
- 서비스명: Orgcell (orgcell.com)
- 핵심 가치: "가족의 역사를 자동으로 정리하고, 안전하게 공유하는 디지털 공간"
- 타겟: 자녀 성장 기록 보관하려는 부모, 가족/친구와 안전하게 사진 공유하려는 사용자

## 2. 차별점 (vs Google Photos / Apple Photos)
1. 쌍방향 AI 사진 교환 (Friend Call) — 기존 서비스에 없는 기능
2. 영지식 E2E 암호화 — 서버 관리자도 사진 못 봄
3. 워터마크 + 소유권 증명 — 도용 방지
4. 독립 가족 전용 웹 공간 — 대형 플랫폼 종속 없음

## 3. MVP 핵심 기능 (Must-Have)
1. 대량 사진 업로드 — pica + Web Workers (브라우저 멈춤 방지)
2. 얼굴 인식 자동 분류 — face-api.js TinyFaceDetector (브라우저 로컬)
3. 보안 갤러리 — TPE (썸네일 보존 암호화, 원본은 암호화 저장)
4. Friend Call — 비밀번호 임시 공유 룸 + 상대 얼굴 필터링
5. 중복 제거 — dHash 지각 해싱
6. PWA — manifest.json + service-worker (오프라인 캐싱, A2HS)
7. 백그라운드 업로드 큐 UI — [얼굴인식 → 암호화 → 업로드] 파이프라인 진행률 표시

## 4. MVP 아키텍처 (비용 최소화)
| 항목         | 원안           | MVP 조정                          |
|-------------|---------------|----------------------------------|
| 스토리지     | S3 직접        | BYOS (Google Drive 연동) → 비용 $0 |
| 소유권 증명  | 블록체인       | 해시 기반 디지털 서명 (SHA-256 + HMAC) |
| P2P 전송    | WebRTC         | 서버 릴레이 (Socket.IO Cloud Buffer) |
| 워터마크     | 동형 암호      | 텍스트 워터마크 (Canvas API)       |

## 5. MVP 제외 (Phase 2+)
- 블록체인 메인넷 연동
- 동형 암호화 워터마킹
- 감정 인식
- 자동 결제 시스템

## 6. 기술 스택
- 프론트엔드: React 18 + Vite + TailwindCSS + pica + face-api.js
- 백엔드: Node.js + Express
- DB: PostgreSQL 16
- 스토리지: Google Drive API (BYOS)
- 통신: Socket.IO 서버 릴레이 (청크 전송)
- 암호화: AES-256-GCM (Web Crypto API), 키 저장 IndexedDB
- EXIF: exifr (DateTimeOriginal, GPS, 카메라 정보)
- 지도: Leaflet + react-leaflet + react-leaflet-cluster
- 인프라: AWS EC2 (t3.small), ECR, Docker Compose
- SSL: Let's Encrypt (orgcell.com)
- CI/CD: GitHub Actions → ECR → EC2 SSH deploy
